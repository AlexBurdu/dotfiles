# Git Commits

- Format: `type: Description` (description starts with uppercase)
- Types: `chore`, `deprecate`, `feat`, `fix`, `release`
- Append `!` for breaking changes (e.g. `fix!: Return type change`)
- Description max 256 characters
- Optional body/footer separated by blank lines
- If working on a tracked issue (e.g. GitHub), reference it in the description (e.g. `fix: Handle null response #42`)
- Do NOT add `Co-Authored-By` lines to commits
